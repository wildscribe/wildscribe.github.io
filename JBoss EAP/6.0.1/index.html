JBoss EAP 6.0.1 Model Reference

  1. JBoss EAP 6.0.1

The root node of the server-level management model.

Children (8)

  • core-service Core services provided by the server.
    • management The management services used to control a standalone server or a host's host controller in a management domain.
    • platform-mbean Provides the management interface for monitoring and management of the Java virtual machine as well as the operating system on which the Java virtual machine is running. Exposes the JDK-provided JMX MBeans in the java.lang, java.nio and java.util.logging JMX domains.
    • server-environment The server environment.
    • service-container The central container that manages all services in a running standalone server or in a host controller in a management domain.
    • vault Security Vault for attributes.
  • deployment A list of deployments available for use on the server.
  • extension A list of extension modules.
  • interface A list of named network interfaces available for use on the server.
  • path A list of named filesystem paths.
  • socket-binding-group A list of socket bindings available for use on the server.
  • subsystem The subsystems that make up the server's profile.
    • cmp The configuration of the cmp subsystem.
    • configadmin The ConfigAdmin subsystem configuration.
    • datasources The data-sources subsystem, used to declare JDBC data-sources
    • deployment-scanner An individual deployment scanner that scans a particular location
    • ee The configuration of the EE subsystem.
    • ejb3 The configuration of the ejb3 subsystem.
    • infinispan The configuration of the infinispan subsystem
    • jacorb The JacORB subsystem configuration.
    • jaxr The configuration of the JAXR subsystem.
    • jaxrs The configuration of the JAX-RS subsystem.
    • jca The Java EE Connector Architecture (JCA) subsystem providing general configuration for resource adapters
    • jdr JBoss Diagnostic Reporter (JDR) Subsystem.
    • jmx The configuration of the JMX subsystem.
    • jpa The configuration of the JPA subsystem.
    • jsr77 The configuration of the JSR-77 subsystem.
    • logging The configuration of the logging subsystem.
    • mail The mail subsystem
    • messaging The messaging subsystem.
    • naming The configuration of the naming subsystem.
    • osgi The OSGi subsystem configuration
    • pojo Support JBoss5 and JBoss6 pojo configuration model.
    • remoting The configuration of the Remoting subsystem.
    • resource-adapters Configuration of the resource adapters subsystem
    • sar The SAR subsystem allowing you to deploy sar archives containing MBeans configured by META-INF/jboss-service.xml
    • security The configuration of the security subsystem.
    • threads The threading subsystem, used to declare manageable thread pools and resources.
    • transactions The configuration of the transaction subsystem.
    • web The configuration of the jboss.web subsystem.
    • webservices The configuration of the web services subsystem.
    • weld The configuration of the weld subsystem.
  • system-property A list of system properties to set on the server.

Precisely one of 'runtime-name', 'hash', 'input-stream-index', 'bytes' or 'url' must be specified.

This would typically be the same as 'name', but in some cases users may wish to have two deployments with the same 'runtime-name' (e.g. two versions of "foo.war") both available in the deployment content repository, in which case the deployments would need to have distinct 'name' values but would have the same 'runtime-name'. false false false

  • resolve-internet-address Takes a set of interface resolution criteria and finds an IP address on the local machine that matches the criteria, or fails if no matching IP address can be found.
    Return Type STRING
    Request Parameter Type Description Nillable Required Expressions Allowed
    any-address BOOLEAN Attribute indicating that sockets using this interface should be bound to a wildcard address. The IPv6 wildcard address (::) will be used unless the java.net.preferIpV4Stack system property is set to true, in which case the IPv4 wildcard address (0.0.0.0) will be used. If a socket is bound to an IPv6 anylocal address on a dual-stack machine, it can accept both IPv6 and IPv4 traffic; if it is bound to an IPv4 (IPv4-mapped) anylocal address, it can only accept IPv4 traffic. true false false
    any-ipv4-address BOOLEAN Attribute indicating that sockets using this interface should be bound to the IPv4 wildcard address (0.0.0.0). true false false
    any-ipv6-address BOOLEAN Attribute indicating that sockets using this interface should be bound to the IPv6 wildcard address (::). true false false
    inet-address STRING Attribute indicating that part of the selection criteria for choosing an IP address for this interface should be whether or not the address matches the given value. Value is either a IP address in IPv6 or IPv4 dotted decimal notation, or a hostname that can be resolved to an IP address. An 'undefined' value means this attribute is not relevant to the IP address selection. true false true
    link-local-address BOOLEAN Attribute indicating that part of the selection criteria for choosing an IP address for this interface should be whether or not the address is link-local. An 'undefined' or 'false' value means this attribute is not relevant to the IP address selection. true false false
    loopback BOOLEAN Attribute indicating that part of the selection criteria for choosing an IP address for this interface should be whether or not it is a loopback address. An 'undefined' or 'false' value means this attribute is not relevant to the IP address selection. true false false
    loopback-address STRING Attribute indicating that the IP address for this interface should be the given value, if a loopback interface exists on the machine. A 'loopback address' may not actually be configured on the machine's loopback interface. Differs from inet-address in that the given value will be used even if no NIC can be found that has the IP specified address associated with it. An 'undefined' or 'false' value means this attribute is not relevant to the IP address selection. true false true
    multicast BOOLEAN Attribute indicating that part of the selection criteria for choosing an IP address for this interface should be whether or not its network interface supports multicast. An 'undefined' or 'false' value means this attribute is not relevant to the IP address selection. true false false
    nic STRING Attribute indicating that part of the selection criteria for choosing an IP address for this interface should be whether its network interface has the given name. The name of a network interface (e.g. eth0, eth1, lo). An 'undefined' value means this attribute is not relevant to the IP address selection. true false true
    nic-match STRING Attribute indicating that part of the selection criteria for choosing an IP address for this interface should be whether its network interface has a name that matches the given regular expression. Value is a regular expression against which the names of the network interfaces available on the machine can be matched to find an acceptable interface. An 'undefined' value means this attribute is not relevant to the IP address selection. true false true
    point-to-point BOOLEAN Attribute indicating that part of the selection criteria for choosing an IP address for this interface should be whether or not its network interface is a point-to-point interface.     site-local-address BOOLEAN Attribute indicating that part of the selection criteria for choosing an IP address for this interface should be whether or it is a site-local address. An 'undefined' or 'false' value means this attribute is not relevant to the IP address selection true false false
    subnet-match STRING Attribute indicating that part of the selection criteria for choosing an IP address for this interface should be whether or it the address fits in the given subnet definition. Value is a network IP address and the number of bits in the address' network prefix, written in "slash notation"; e.g. "192.168.0.0/16". An 'undefined' value means this attribute is not relevant to the IP address selection. true false true
    up BOOLEAN Attribute indicating that part of the selection criteria for choosing an IP address for this interface should be whether its network interface is currently up. An 'undefined' or 'false' value means this attribute is not relevant to the IP address selection true false false
    virtual BOOLEAN Attribute indicating that part of the selection criteria for choosing an IP address for this interface should be whether its network interface is a virtual interface. The value of the attribute is a set of criteria. An 'undefined' value means this attribute is not relevant to the IP address selection. true false false
