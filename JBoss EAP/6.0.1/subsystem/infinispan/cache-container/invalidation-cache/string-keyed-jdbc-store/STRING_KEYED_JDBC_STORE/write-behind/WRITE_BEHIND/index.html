JBoss EAP 6.0.1 Model Reference

  1. JBoss EAP 6.0.1
  2. subsystem=infinispan
  3. cache-container
  4. invalidation-cache
  5. string-keyed-jdbc-store=STRING_KEYED_JDBC_STORE
  6. write-behind=WRITE_BEHIND

Configures a cache store as write-behind instead of write-through.

Attributes (4)

  • flush-lock-timeout Timeout to acquire the lock which guards the state to be flushed to the cache store periodically.
    • Attribute Value
    Default Value 1
    Type LONG
    Nillable true
    Expressions Allowed false
  • modification-queue-size Maximum number of entries in the asynchronous queue. When the queue is full, the store becomes write-through until it can accept new entries.
    • Attribute Value
    Default Value 1024
    Type INT
    Nillable true
    Expressions Allowed false
  • shutdown-timeout Timeout in milliseconds to stop the cache store.
    • Attribute Value
    Default Value 25000
    Type LONG
    Nillable true
    Expressions Allowed false
  • thread-pool-size Size of the thread pool whose threads are responsible for applying the modifications to the cache store.
    • Attribute Value
    Default Value 1
    Type INT
    Nillable true
    Expressions Allowed false
