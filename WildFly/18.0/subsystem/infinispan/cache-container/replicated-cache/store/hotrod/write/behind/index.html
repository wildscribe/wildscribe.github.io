  6. write=behind

Configures a cache store as write-behind instead of write-through.

Attributes (4)

  • flush-lock-timeout Timeout to acquire the lock which guards the state to be flushed to the cache store periodically.
    • Deprecated Since 4.0.0
    Deprecated. This attribute is no longer used.
    Attribute Value
    Default Value 5000
    Type LONG
    Nillable true
    Expressions Allowed true
    Unit MILLISECONDS
    Storage configuration
    Access Type read-write
    Restart Required resource-services
  • modification-queue-size Maximum number of entries in the asynchronous queue. When the queue is full, the store becomes write-through until it can accept new entries.
    • Attribute Value
    Default Value 1024
    Type INT
    Nillable true
    Expressions Allowed true
    Storage configuration
    Access Type read-write
    Restart Required resource-services
  • shutdown-timeout Timeout in milliseconds to stop the cache store.
    • Deprecated Since 4.0.0
    Deprecated. This attribute is no longer used.
    Attribute Value
    Default Value 25000
    Type LONG
    Nillable true
    Expressions Allowed true
    Unit MILLISECONDS
    Storage configuration
    Access Type read-write
    Restart Required resource-services
  • thread-pool-size Size of the thread pool whose threads are responsible for applying the modifications to the cache store.
    • Attribute Value
    Default Value 1
    Type INT
    Nillable true
    Expressions Allowed true
    Storage configuration
    Access Type read-write
    Restart Required resource-services

Operations (2)

  • add Adds a write-behind configuration element to the store.
    Request Parameter Type Required Expressions Allowed Default value Description
    modification-queue-size INT false true 1024 Maximum number of entries in the asynchronous queue. When the queue is full, the store becomes write-through until it can accept new entries.
    thread-pool-size INT false true 1 Size of the thread pool whose threads are responsible for applying the modifications to the cache store.
    flush-lock-timeout LONG false true 5000 Timeout to acquire the lock which guards the state to be flushed to the cache store periodically.
    shutdown-timeout LONG false true 25000 Timeout in milliseconds to stop the cache store.
  • remove Removes a write-behind configuration element from the store.
