A broadcast group is the means by which a server broadcasts connectors over the network.

Attributes (9)

  • broadcast-period The period in milliseconds between consecutive broadcasts.
    • Attribute Value
    Default Value 2000
    Type LONG
    Nillable true
    Expressions Allowed true
    Unit MILLISECONDS
    Storage configuration
    Access Type read-write
    Restart Required all-services
  • connectors Specifies the names of connectors that will be broadcast.
    • Attribute Value
    Type LIST
    Nillable true
    Expressions Allowed false
    Storage configuration
    Access Type read-write
    Restart Required all-services
  • group-address Deprecated. The multicast address to which the data will be broadcast. It is a class D IP address in the range 224.0.0.0 to 239.255.255.255, inclusive. The address 224.0.0.0 is reserved and is not available for use.
    • Deprecated Since 1.2.0
    Use instead a socket-binding to specify the broadcast group's group address.
    Attribute Value
    Type STRING
    Nillable true

    Access Type read-write
    Restart Required all-services

    • Attribute Value
    Type STRING
    Nillable true
    Expressions Allowed true
    Storage configuration
    Access Type read-write
    Restart Required all-services
  • local-bind-address Deprecated. The local bind address that the datagram socket is bound to. If you have multiple network interfaces on your server, you would specify which one you wish to use for broadcasts by setting this attribute. If this attribute is not specified then the socket will be bound to the wildcard address, an IP address chosen by the kernel.
    • Deprecated Since 1.2.0
    Use instead a socket-binding to specify the broadcast group's local bind address.
    Attribute Value
    Type STRING
    Nillable true
    Expressions Allowed false
    Storage configuration
    Access Type read-write
    Restart Required all-services
  • local-bind-port Deprecated. The local port to which the datagram socket is bound. Normally you would just use the default value of -1 which signifies that an anonymous port should be used. This parameter is always specified in conjunction with local-bind-address.
